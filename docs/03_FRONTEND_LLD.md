# 03 — Frontend Low-Level Design

> **Audience.** Engineer/agent implementing the React SPA. Read `00_MASTER_PLAN.md` and `01_HLD.md` first.
>
> **North star.** A premium, accessible, dark-first UI that lets a non-expert understand a voice-based Parkinson's prediction in 60 seconds, and lets an expert dig as deep as they want.
>
> **Acceptance.** Lighthouse ≥ 90 across all four categories on the Predict page. End-to-end test "record → predict → download PDF" passes in Playwright.

---

## 3.1 Tech Stack (locked)

See `01_HLD.md` §1.3 for full version pins. Headlines:

- React 18, TypeScript 5.5 strict, Vite 5
- Tailwind CSS + shadcn/ui (Radix primitives copied into our repo)
- TanStack Query v5 for all server state, Zustand for tiny slice of client state
- React Hook Form + Zod for forms (Zod schemas mirror backend Pydantic)
- React Router v6
- Recharts for "common" charts; visx for SHAP / heatmap / PCA scatter
- Framer Motion for micro-interactions (respects `prefers-reduced-motion`)
- `@react-pdf/renderer` for PDF export (client-side)
- `openapi-typescript-codegen` to generate the API client + types from `/api/v1/openapi.json`
- Vitest + Testing Library for unit, Playwright for E2E
- ESLint (`@typescript-eslint`, `react-hooks`, `jsx-a11y`) + Prettier + `lint-staged`

**Conventions**
- One component per file, named exported, file name matches component.
- Server state: TanStack Query. Client state: Zustand. **Never** put server data in Zustand.

- Every interactive element passes `aria-*` and is reachable via keyboard.

### 3.3.2 Typography
- Base font Inter (variable). Self-hosted via `@fontsource-variable/inter`. **No Google Fonts request** (CSP friendly, faster).
- Numerals: tabular nums for all metric displays (`font-variant-numeric: tabular-nums`).

### 3.3.3 Iconography
- `lucide-react` (tree-shaken). One import per icon. Stroke 1.5.

### 3.3.4 Motion
- Default duration 150 ms ease-out for hover/focus, 250 ms ease-out for entrance, 400 ms for page transitions.
- Wrap any `motion.*` component with `useReducedMotion()` early-return.

### 3.3.5 Color semantics for predictions
- **Healthy** (label 0): success-600 fill, success-50 background.
- **Parkinson's predicted** (label 1, p ≥ 0.7): danger-600 fill, danger-50 bg.
- **Borderline** (0.4 ≤ p < 0.7): warning-600 fill, warning-50 bg.
- These three classes are computed in one helper `verdictTone(p: number)` and reused everywhere.

## 3.5 Page Specifications

> All pages share `<AppShell>` which renders `<TopNav>`, an `<Outlet>`, the persistent `<DisclaimerBanner>`, and `<Footer>`.

### 3.5.1 Home (`/`)
**Goal:** clarify what this is in 5 seconds, then route the user.

Sections (top to bottom):
1. **Hero** — headline ("See what your voice can suggest about Parkinson's disease — *for science, not diagnosis*"), two primary CTAs: **"Try with my voice"** → `/audio`, **"Enter features manually"** → `/predict`. Sub-CTA: **"Explore the data"** → `/explorer`.
2. **Trust strip** — three cards: "9 ML models", "SHAP explanations", "Open data".
3. **How it works** — 3-step illustrated explanation.
4. **Live mini-stats** — pulled from `/api/v1/analytics/dataset-stats`: total samples, class balance, accuracy of best model.
5. **About the science** card → `/about`.
6. Footer with disclaimer + links.

Animation: hero text fades in 200 ms, stats count up via `react-countup` (respects reduced motion).

### 3.5.2 Predict — manual (`/predict`)
**Layout**: two-column on `lg+`, single column below.

**Left column — Feature form (`<FeatureForm>`):**
- Grouped accordions (default: first group open):
  - Frequency (`MDVP:Fo/Fhi/Flo(Hz)`)
  - Jitter (5 fields)
  - Shimmer (6 fields)
  - Harmonicity (NHR, HNR)
  - Nonlinear (RPDE, DFA, spread1/2, D2, PPE)
- Each field renders `<FeatureWithReference>`: numeric input (with up/down step buttons), unit chip, color-coded reference range badge (green/amber/red based on dataset percentiles), tooltip on the label with a 1–2 sentence definition + citation.
- Top toolbar: **Sample loader** dropdown ("Random", "Typical Healthy", "Typical Parkinson's"); **Reset**; **Model** select (defaults to "Auto — best of all").
- Submit button shows a spinner; disabled while invalid.
- Validation via Zod mirroring backend ranges; inline errors under each field; toast on submit error.
- Keyboard: `Cmd/Ctrl+Enter` submits; `?` opens shortcuts modal.

**Right column — `<ResultCard>` (renders into a slot, sticky on `lg+`):**
- Header: verdict text ("Likely **Parkinson's**", "Likely **healthy**", "Borderline") with tone color.
- `<ProbabilityGauge>` (semicircle, visx arc): primary model's probability, animated from 0 to value over 600 ms.
- `<PerModelStrip>`: 9 chips (KNN, SVM, ...) each colored by their probability and labeled with `pp%`. Hover reveals exact value + version.
- `<ShapWaterfall>`: top-5 features by `|shap|`, sorted, with bars in tone color. Toggle "show all 22" reveals full waterfall.
- Action row:
  - **Download report (PDF)** — opens `<ReportDocument>` PDF in a new tab.
  - **Copy share link** — encodes features into URL, copies to clipboard.
  - **What if?** — opens `<WhatIfPanel>` (drawer).
  - **Send feedback** — small thumbs up/down.

Empty state (no submission yet): show a 3-tile preview of what they'll get + a "Try a sample" button.

### 3.5.3 Predict — audio (`/audio`)
**Layout:** centered single column.

Steps:
1. **Consent block** — checkbox: "I understand this is research, not a diagnosis." (Stored in `consent` Zustand slice; reused across pages.)
2. **Recorder** (`<Recorder>`):
   - Big circular record button (Radix toggle). States: idle / requesting permission / recording / processing.
   - Live `<Waveform>` (AnalyserNode → 32 bars).
   - Timer 0:00 / 0:30. Auto-stop at 30 s, warns at 25.
   - Re-record / Use this clip buttons after stop.
3. **Upload alternative** — drag-and-drop or "Choose file" (`audio/wav`, `audio/webm`, `audio/flac`, ≤ 5 MB).
4. **Optional: Save audio for research** toggle (`<ConsentToggle>`) → sets `Consent-Store-Audio: true`.
5. On submit: shows `<ExtractedFeatures>` (read-only mirror of the 22 features the server extracted, color-coded) **plus** `<ResultCard>` from §3.5.2.

Errors:
- Permission denied → friendly explainer + link to manual mode.
- iOS Safari quirks → fallback to file upload only with note.

Performance: chunked upload aborts if `> 5 MB` is reached client-side before sending.

### 3.5.4 Batch (`/batch`)
- `<Uploader>` accepts CSV with the 22 columns (header validated client-side; first error highlighted).
- Sample CSV download link.
- After upload → `<JobStatusCard>` polls `/api/v1/batch/{id}` every 2 s with exponential backoff cap at 10 s.
- On success → `<ResultsTable>` (TanStack Table), virtualized; sortable; filter "Predicted Parkinson's only"; "Download CSV" button.
- Job retention note: "Results auto-delete after 24 hours."

### 3.5.5 Dashboard (`/dashboard`)
Tabs (Radix Tabs): **Compare**, **Confusion Matrices**, **ROC**, **Precision/Recall**, **Calibration**.

- **Compare**: bar chart of all 9 models on 5 metrics; sortable. Below: a table view with sparkline of CV stability (mean ± std).
- **Confusion Matrices**: grid of 9 small heatmaps; click any to enlarge in a modal.
- **ROC / PR / Calibration**: line charts (Recharts), one line per model, click legend to toggle.

Each chart has an "Explain this chart" disclosure with a 2-sentence explanation aimed at a medical student.

### 3.5.6 Explorer (`/explorer`)
- `<PcaScatter>` colored by status; brushable to filter the dataset table below.
- `<CorrelationHeatmap>` 22×22; click cell → opens detail modal with the bivariate scatter.
- `<FeatureDistribution>` for any chosen feature: histogram + per-class boxplot.
- `<DatasetTable>` virtualized; export filtered subset as CSV.

### 3.5.7 What-If (drawer, reachable from `/predict` and `/audio`)
- Lists all 22 features with sliders (min/max from dataset percentiles).
- Drag a slider → debounced (250 ms) re-call to `/predict` with the modified feature; the gauge animates between values.
- "Reset to original" and "Apply to form" buttons.

### 3.5.8 About (`/about`)
- Plain prose: project goal, dataset description, references (with the IEEE paper from `README.md`), team / acknowledgements.
- Glossary table for the 22 features (definition, reference, units).
- "Limitations" subsection — explicit about dataset size, class imbalance, single-language, demographic bias.

### 3.5.9 Admin (`/admin/*`)
- Login form posts to `/auth/login`; on success redirects to `/admin`.
- `/admin` shows tabs: **Predictions**, **Feedback**, **Batch jobs**, **Audit log**, **Models** (lists manifest with metrics + "trigger retrain" button).
- All list views use cursor pagination; none of the admin pages are indexed (`<meta name="robots" content="noindex">`).

## 3.6 State Management

| State | Where | Why |
|---|---|---|
| Server data | TanStack Query | Request dedup, cache, retry, suspense. |
| Theme | Zustand (persisted to `localStorage`) | Tiny, sync, no provider hell. |
| Recent predictions (last 10) | Zustand (persisted) | Cross-page recall without backend round-trip. |
| Consent flags | Zustand (persisted) | Need to gate UI. |
| Form values | React Hook Form (per-form local) | Avoid global form state. |
| Auth (admin token presence) | TanStack Query (`useQuery(['me'])`) | Server is source of truth. |

QueryClient defaults:
```ts
const queryClient = new QueryClient({
  defaultOptions: {
    queries: { staleTime: 30_000, gcTime: 5 * 60_000, retry: 1, refetchOnWindowFocus: false },
    mutations: { retry: 0 },
  },
});
```

Query keys are centralized in `src/api/queryKeys.ts`:
```ts
export const qk = {
  models: () => ['models'] as const,
  modelMetric: (name: string, metric: string) => ['models', name, metric] as const,
  analytics: {
    dataset: () => ['analytics', 'dataset'] as const,
    correlation: () => ['analytics', 'correlation'] as const,
    pca: () => ['analytics', 'pca'] as const,
```

## 3.7 Forms

Every form is built with React Hook Form + Zod. The Zod schemas live in `src/lib/schemas/` and are derived (by hand, kept in sync with backend Pydantic via a snapshot test that diffs against the OpenAPI schema):

```ts
// src/lib/schemas/voiceFeatures.ts
import { z } from 'zod';
export const VoiceFeaturesSchema = z.object({
  'MDVP:Fo(Hz)':       z.number().min(50).max(300),
  'MDVP:Fhi(Hz)':      z.number().min(50).max(600),
  // ... all 22 with the same bounds as backend
});
export type VoiceFeaturesInput = z.infer<typeof VoiceFeaturesSchema>;
```

Form pattern:
```tsx
const form = useForm<VoiceFeaturesInput>({
  resolver: zodResolver(VoiceFeaturesSchema),
  mode: 'onBlur',

const onSubmit = form.handleSubmit((values) =>
  predictMutation.mutate({ features: values })

```

Errors render in `<FormError>` under each field. A `<FormSummary>` shows count of errors at top of the form.

## 3.9 Audio Recording (`src/lib/audio/recorder.ts`)

```ts
export class Recorder {
  private stream?: MediaStream;
  private rec?: MediaRecorder;
  private chunks: BlobPart[] = [];
  async start() {
    this.stream = await navigator.mediaDevices.getUserMedia({ audio: { channelCount: 1, sampleRate: 22050 }});
    const mime = MediaRecorder.isTypeSupported('audio/webm;codecs=opus')
      ? 'audio/webm;codecs=opus' : 'audio/webm';
    this.rec = new MediaRecorder(this.stream, { mimeType: mime, audioBitsPerSecond: 64_000 });
    this.rec.ondataavailable = (e) => e.data.size && this.chunks.push(e.data);
    this.rec.start(250);
  }
  async stop(): Promise<Blob> {
    return new Promise((resolve) => {
      if (!this.rec) throw new Error('Not recording');
      this.rec.onstop = () => {
        const blob = new Blob(this.chunks, { type: this.rec!.mimeType });
        this.stream?.getTracks().forEach((t) => t.stop());
        resolve(blob);
      };
      this.rec.stop();
    });
  }
}
```

Waveform uses `AudioContext`'s `AnalyserNode` `getByteFrequencyData` at 60 fps (skipped if reduced motion).

## 3.12 Accessibility

- All inputs have visible labels (no placeholder-as-label).
- Color is never the sole signal: every verdict has an icon + text label.
- Min target size 44×44 for primary actions on touch.
- Focus styles: `outline-2 outline-offset-2 outline-focus`. Never remove outline without replacement.
- `aria-live="polite"` region for the result card so screen readers announce updates.
- Keyboard: tab order matches reading order; modals trap focus (Radix gives this for free).
- `prefers-reduced-motion`: skip mount animations; keep duration-zero variants for layout shifts.
- Lighthouse accessibility ≥ 95; axe-core in CI.

## 3.14 Performance

- Route-level code splitting (lazy + Suspense fallback skeletons).
- Bundle budgets enforced by `vite-bundle-visualizer` in CI: initial ≤ 250 KB gzipped, Predict route ≤ 80 KB extra.
- Images SVG or WebP; `<img>` with `loading="lazy"` outside the fold.
- TanStack Query staleTime 30 s reduces re-fetches.
- `react-virtual` for any list ≥ 100 rows.
- Memoization only when profiler demonstrates need.

---

## 3.15 Error Handling

- `<ErrorBoundary>` at the App level + per-route boundary; shows `<ErrorState>` with the request ID.
- Toast for transient errors (use `sonner` or shadcn's toast); 5 s auto-dismiss; error toasts persist until dismissed.
- All mutations show optimistic UI where safe (e.g., disabling form, showing pending state); rollback on error.
